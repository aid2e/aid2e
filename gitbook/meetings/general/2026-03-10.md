# AID2E bi-weekly general meeting

**Meeting:** March 10, 2026

**Attendees:** Tianle Wang, Kolja Kauder, Baptiste Fraisse, Anselm Vossen, Derek Anderson, Hemalata Nayak, Karthik Suresh, Cristiano Fanelli

* Announcements
  * CHEP remained an active planning topic.
* Phase I
  * The group reviewed the paper draft in [Overleaf](https://www.overleaf.com/project/69aef923481e484bcf40ff62).
  * CPC submission requirements were checked against the [Computer Physics Communications author guide](https://www.sciencedirect.com/journal/computer-physics-communications/publish/guide-for-authors).
  * The team noted that CPC expects source code, a README, installation and execution instructions, and at least one test dataset with sample output.
  * The website and meeting-note structure were also discussed, with a template planned for the following week.
  * The framework was originally expected by the end of February, but the group agreed more work was still needed.
  * Software compatibility issues had recently been addressed through a merge.
  * The scheduler was described as working, but the next technical meeting should include a concrete running example.
  * Duke was identified as one likely source for the first dRICH-based example.
  * Documentation and a clearer near-term timeline remained immediate Phase I priorities.
* Phase II
  * The group reviewed recent AI4EIC activity around RAG systems as a possible alignment point for AID2E.
  * The [RAG4EIC meeting agenda](https://docs.google.com/document/d/19kdKaoZlwWSqzpv81jOVGZfv3jO4h37v_eab9ltYVhI/edit?usp=sharing) was referenced.
  * The [RAG4EIC project](https://rag4eic.ds.wm.edu/%20https://github.com/ai4eic/EIC-RAG-Project) was noted as a relevant external effort.
  * BNL activity in that space was highlighted.
  * The group discussed whether RAG-style tooling could help with logging, monitoring, and cross-runner support.
  * ePIC remained the most natural use case for that Phase II direction.
